# My Solution

`-S` 选项支持 `SWITCH_ON_END` 和 `SWITCH_ON_IO`。前者在进程结束时切换进程，后者在进程申请 IO 操作时切换进程。

在进程申请 IO 时切换进程往往是一个更好的选择。因为 IO 操作往往花费很长时间，这个等待时间让给其他进程操作正合适。

---

`-I` 选项支持 `IO_RUN_LATER` 和 `IO_RUN_IMMEDIATE` 。前者指示在 IO 操作完成后该进程等待其他进程完成后再执行，后者指示 IO 操作完成后立刻执行该进程。

后者的效率似乎更高。因为我们可以相信一个调用过 IO 的进程更有可能下一步也需要进行 IO 操作。当现在的 IO 操作完成后立刻继续运行这个进程，它可能很快又会触发下一个 IO 操作，其他进程就可以利用这段时间处理。而如果是前者这个策略，那么当前进程 IO 操作结束后，就必须等待其他进程结束才能开始它下一个 IO 操作，浪费了时间。
